# Brainstorming

## Overview
The brainstorming skill enables structured exploration of ideas and design solutions through systematic inquiry and collaborative thinking.

## When to Use

- User wants to explore multiple approaches to a problem
- Designing a new feature or system
- Thinking through complex decisions
- User explicitly asks "what are my options?" or "how should I approach this?"

## Process

1. **Explore Context** - Understand the project, constraints, and goals
2. **Clarify Requirements** - Ask one clarifying question at a time
3. **Generate Options** - Propose 2-3 approaches with trade-offs
4. **Present Design** - Use visual aids if helpful
5. **Document Decision** - Write a brief design doc for user approval

## Hard Gate

Do NOT invoke any implementation skill until the user has approved a design.

## Rules

- Ask ONE question at a time
- Present trade-offs clearly for each option
- Be willing to say "I don't know, let me explore"
- Don't commit to approaches before understanding the full context
- Use visual representations when they aid understanding
